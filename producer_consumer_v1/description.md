# Простая задача производителя-потребителя на wait() и notify().
# Один производитель - один потребитель

Massage Producer - создает и передает сообщения в Message Broker.
Message Broker - складирует переданные сообщения.
Message Consumer - потребляет сообщения с Message Broker и удаляет сообщения, которые уже принял.

У нас должно быть запущено два потока.
Consuming Thread - потребляет сообщения из брокера
Producing Thread - кладет сообщения в брокер

Также брокер хранит максимум 5 сообщений.

Так как два потока параллельно работают с очередью сообщений брокера, эту работу нужно синхронизировать.
Будем использовать ключевое слово synhronized.

У брокера будет два НЕСТАТИЧЕСКИХ метода.
consume() - вызывается потоком Consuming thread
produce() - вызывается потоком Producing thread
То есть потоки будут синхронизироваться по экземпляру MessageBroker

wait() и notify() могут вызываться только внутри синхронизированных методов или синхронизированных блоков.
